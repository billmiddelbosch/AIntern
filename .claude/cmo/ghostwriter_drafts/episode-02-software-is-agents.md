# Episode 2 — Software bouwen is agents bouwen.

**Draft:**

𝗢𝗻𝘀 𝗔𝗜-𝘀𝘆𝘀𝘁𝗲𝗲𝗺 𝗸𝗼𝘀𝘁𝘁𝗲 €𝟵𝟵,𝟲𝟵 𝗽𝗲𝗿 𝘄𝗲𝗲𝗸. 𝗗𝗲 𝗼𝗺𝘇𝗲𝘁 𝘄𝗮𝘀 €𝟬.

Dat was de samenvatting van week 16.

We bouwden een AI die taken uitvoert — en betaalden per token die hij gebruikte. Elke boardvergadering, elke draft, elke keer dat een agent nadacht: kosten. Geen inkomsten.

De week erna installeerden we RTK. Een tool die de Claude-output comprimt voordat die terugkomt in de context. Commit `9dcc05c`, 19 april. Projectie: 60–90% minder tokens op build- en testoutput.

Maar dat was niet het echte inzicht.

Het echte inzicht was wat we precies aan het bouwen waren. Geen features. Geen knoppen. We schreven instructiesets: `.claude/skills/lead-outreach.md`. `.claude/agents/ghostwriter.md`. Bestanden die beschrijven hoe een agent denkt, wat hij mag beslissen, wanneer hij moet stoppen en vragen.

Gedrag definiëren is moeilijker dan code schrijven.

Want code doet wat je schrijft. Een agent doet wat hij begrijpt. En als je hem niet precies genoeg instrueert, besluit hij iets wat je niet had bedoeld — en schrijft hij een LinkedIn-post die meer op een advertentie lijkt dan op een verhaal.

Elke fout werd een zin in een skill-file.

Zo bouw je agents. En zo leer je wat ze eigenlijk zijn.
